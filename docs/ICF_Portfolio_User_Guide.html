What is this?

The ICF Creative Portfolio is a desktop app for browsing the team's creative work — every project, every banner, every postcard, organized by client and program. It lives on your Mac. It reads files from Box. And it stays current automatically.

Why a desktop app and not just a website?

Because the work lives in Box, and Box's online-only files don't load well in a regular browser. The desktop app is a thin wrapper around the same portfolio you'd see in a browser — but it can read files directly from your local Box mount, including PDFs, large images, and videos, without choking on Box's quirky permissions.

What you can do with it

  • Browse every project from every client in one place
  • Switch between Cards, Matrix, List, and Tiles views
  • Group by program or by month
  • Filter by asset type (emails, banners, postcards, social, etc.)
  • Search across all clients
  • Compare projects side-by-side
  • Open PDFs and images in their native viewers

Installation

First-time setup takes about 60 seconds.

Step 1 — Get the DMG

The current installer lives in our shared Box folder:

Box / Clients / BGE / portfolio-master / ICF Creative Portfolio-X.Y.Z-arm64.dmg

If your teammate sent you a Box link, click it and download the DMG.

Tip: Drag the DMG to your Desktop before opening it. Opening DMGs directly from inside Box can fail because Box may lock the file mid-mount.

Step 2 — Install

1

Double-click the DMG

A window opens showing the app icon and an Applications shortcut.

2

Drag ICF Creative Portfolio onto the Applications shortcut

It copies to your /Applications folder.

3

Eject the DMG

Right-click the mounted disk on your desktop → Eject.

4

Launch from Applications

Find "ICF Creative Portfolio" in /Applications and double-click. macOS may ask if you're sure you want to open it — yes, you are.

Step 3 — First-launch setup

On first launch, the app asks where your Box folder lives. The default is usually correct:

~/Library/CloudStorage/Box-Box

If the app auto-detects this and finds the portfolio inside, you won't see a picker at all — the window just opens to the portfolio.

If you see a folder picker, navigate to your Box-Box folder and click Open. The app remembers this choice for future launches; you can change it later from Settings.

If the app hangs on launch with no window: your Box folder may still be downloading files in the background after a Box Drive reset. Wait a few minutes and try again. If it still hangs, see Troubleshooting at the end of this guide.

Using the app

The sidebar

The left sidebar has:

  • Client dropdown at the top — switch between BGE, SMECO, PSE&G, PNM, etc.
  • Search bar — type to filter projects across the whole client by name or job number.
  • BY TYPE dropdown — narrow to just emails, banners, postcards, etc.
  • Notes — select a note to view shared review-meeting notes for this client, or click + New Note to create one.
  • Project list — clickable links to each project. Groups by Program or Month depending on your settings.
  • ↻ Refresh at the bottom — pulls the latest builds. The app will also prompt you with a toast notification whenever new work arrives.
  • ⚙ Settings — opens the Settings panel (next section).

The main viewer

Click any project to load it. The default layout is Matrix — a grid of thumbnails. You can switch layouts at any time using the toolbar above the project:

  • Cards — large thumbnails with metadata
  • Matrix — tight grid of thumbnails (default)
  • List — compact rows
  • Tiles — variable-size tiles

Click any thumbnail to open the full asset. PDFs and PNGs open in your default browser; office docs (Word, Excel) open in a preview panel.

Compare mode

Click the Compare button in the toolbar to put two projects side-by-side. Each side has its own client/year/program/project dropdowns so you can mix and match across clients.

Shared Notes

In the sidebar, under the Notes section, you can view, add, and edit shared notes for the current client. This is useful for storing review-meeting notes or campaign guidelines.

  • Notes are saved to the shared Box folder and are instantly visible to the whole team.
  • If the + New Note button is hidden, it means your local API server isn't running (see the Server mode section). Turn on Server mode to add or edit notes.

Settings

Open Settings by clicking ⚙ Settings at the bottom of the sidebar. There are four tabs:

Clients tab

Shows every client the portfolio knows about, with job counts and last-build time. Built-in clients (BGE, SMECO, PSE&G, PNM) can't be deleted; user-added clients can be edited or removed.

Click + Add Client to register a new client with its own logo, color, and folder mapping.

Server tab Advanced

This is where you opt in to being the team's "build server" — the one machine that watches Box for new files and rebuilds the portfolio for everyone. Most people leave this off. See the next section for how this works.

About tab

Shows your app version, your Box folder, and a Check for updates button. The app now auto-checks for updates shortly after launch, then downloads in the background and shows a Restart to update prompt when it's ready. The button is still there if you want to manually retry the check. If your network can't reach GitHub, install manually from the Box DMG instead.

Server mode Optional

By default, your copy of the app is read-only. Someone, somewhere, has to actually run the build pipeline that scans Box for new creative work and refreshes the portfolio. That someone is whoever flips on Server mode.

How it works

When you toggle Run build server on this machine in Settings → Server:

  • The app spawns a Python build process in the background. It scans Box for new projects every 60 seconds (or every 3 minutes if fswatch isn't installed).
  • The app spawns a small local HTTP server on localhost:8765 for API endpoints (used by the in-app Settings panel to edit clients).
  • The app writes a heartbeat to Clients/BGE/portfolio-master/.builder-active.json every 30 seconds so other people's apps know that you're "active."

Who should run it?

Just one person at a time. The status pill at the top of the Server tab tells you what's happening:

Idle No one is currently running the server. If you flip it on, you become the active builder.
Active: someone-else's Mac Someone else is already running it. You don't need to. (If you flip it on anyway, the app will ask if you want to take over.)
Running on this Mac You are the active builder. Leave the app running.

When the active builder goes offline

If the previous active builder hasn't sent a heartbeat in over 2 minutes (laptop closed, internet down, app quit), their lock becomes "stale." Anyone else can take over without prompting.

This means: if Alex closes his laptop at 5pm and goes home, and a teammate needs to push a new build at 7pm, they can just open the app, flip on Server mode, and it works — no manual handoff.

What it costs you

The build process is CPU-light when there's nothing to do (it just checks file timestamps). A full rebuild after many new files takes 1–3 minutes and uses some RAM. You can leave it running all day with no noticeable impact on other work.

A note about the Clients and Notes features

The Clients tab in Settings (where you add/edit/delete clients) and the Notes feature in the sidebar both need a small API to be running on localhost:8765. That API is part of the server pipeline. So: if the in-app server toggle is OFF, the Clients tab and Notes feature will be in read-only mode (the "+ New Note" button will be hidden). If you need to edit a client or add a note, toggle the server on first, make your edits, then optionally toggle it off again.

Updates

The app now checks GitHub for updates automatically after it opens. If that background check can't reach GitHub, new versions still land in Box and can be installed manually.

How you'll know there's a new version

The current DMG in Box has the version number in its filename — e.g. ICF Creative Portfolio-0.3.0-arm64.dmg. Compare it to the version shown in Settings → About → App version. If Box has a higher number, install the newer DMG.

Why keep the Box DMG around? The app now auto-checks safely after the window finishes loading, but some ICF Macs still may not be able to reach GitHub Releases. The shared Box DMG remains the fallback path for those machines.

Installing a newer version (over the top)

1

Quit the running app

⌘Q from the menu bar. If it's not responding, force-quit (⌥⌘Esc).

2

Drag the new DMG out of Box to your Desktop

Always copy to your Desktop first — opening DMGs from inside Box occasionally fails.

3

Open the DMG, drag the app to /Applications

macOS will ask if you want to replace the existing app. Say yes.

4

Eject the DMG, launch the new version from Applications

Your settings, Box folder selection, and any local data are preserved.

The "Check for updates" button

The app checks automatically on launch, but there's still a Check for updates button in Settings → About. Use it if you want to retry immediately. If your network allows GitHub, the app will download the new version and prompt you to restart. If it just spins or never finishes, your network is blocking GitHub — fall back to the manual Box install instead.

If something looks broken after an update

The most common issue after upgrading is a partial install state. To fully reset:

1. Quit the app (⌘Q)
2. Drag /Applications/ICF Creative Portfolio.app to Trash
3. Open Finder → press ⌘⇧G → paste:
   ~/Library/Application Support/ICF Creative Portfolio
4. Drag that folder to Trash
5. Empty Trash
6. Reinstall from the Box DMG

This wipes your local app preferences (Box folder selection, layout choices) and gives you a fully clean install. You'll re-pick your Box folder on first launch.

Troubleshooting

The app opens but shows a blank window — empty client dropdown, no projects

This is almost always Box's "online-only" file stubs. Box doesn't actually download files until you access them; when you first install, the manifest files (clients.js, manifest-*.js) may not be on your Mac yet.

Fix:

  1. Quit the app.
  2. In Finder, navigate to Box-Box / Clients / BGE / portfolio-master.
  3. Right-click the portfolio-master folder → "Make Available Offline" (or "Always Keep On This Device").
  4. Wait 1–3 minutes for the cloud-icon overlays to disappear from the files inside.
  5. Reopen the app.

You only need to do this once. After that, Box keeps the folder synced automatically.

The app won't open / hangs without showing a window

This was a bug in older versions. If you're still seeing it, you have an outdated version. Get the latest DMG from Box and reinstall (see Updates section above).

The app says "That folder doesn't look like your Box folder"

Box hasn't finished syncing yet, or the team's portfolio folder hasn't been shared with you. Confirm you can navigate to Box-Box / Clients / BGE / portfolio-master in Finder and that you see an index.html file inside. If you don't, ask whoever owns the Box shared link to re-invite you.

Some projects show "No assets" even though I know there are files

Two possible reasons:

  • Box hasn't downloaded those files yet. Box uses "online-only" stubs for files you haven't opened. The build script can only see files that exist locally. Right-click the project folder in Finder → Make Available Offline to force-download it. Wait a minute, then click Refresh in the portfolio sidebar.
  • The build hasn't run since those files arrived. Click ↻ Refresh in the sidebar. If the active builder is offline, ask someone to flip on Server mode.

Settings → Clients tab shows "read-only mode"

That means there's no local API server running. The Clients tab uses a small Python server on localhost:8765 to add/edit/delete clients. To enable editing: Settings → Server → flip "Run build server on this machine" ON. The pill should turn green within a few seconds, and the Clients tab will become editable.

If you don't intend to run the server, the Clients tab will remain read-only — that's normal. Only the person designated as the active builder needs the toggle on.

"Check for updates" button just spins forever

Your network is blocking GitHub (some ICF corporate Macs do this). That's fine — the app will still work normally. Just fall back to manual installs from Box when you need a newer version.

The auto-refresh toast keeps appearing

That means the build server is actively republishing the portfolio. Click Refresh and it'll go away.

I see a sync conflict / .conflict file in the portfolio folder

Two people accidentally ran the build server at the same time. The heartbeat lock should prevent this, but if you see conflict files:

  1. Coordinate with the team to make sure only one person has Server mode on.
  2. Delete the .conflict files in Clients/BGE/portfolio-master/.
  3. Wait for the next build to overwrite cleanly.

I think there's a bug

Note what you did, what you expected, and what happened. Tell Alex or whoever is currently maintaining the app. Bonus points for a screenshot.

The 30-second mental model

  • App look & feel → edit index.html in Box. Reload the app with ⌘R. Done.
  • App behavior / Electron logic → edit main.js or preload.js, then rebuild + notarize + publish a new release. The build guide has the exact commands.
  • Build engine / what gets thumbnailed and indexed → edit build-portfolio.py, copy into v0.1.0/py-scripts/, then rebuild + publish.
  • Auto-update channel → publish via gh release create. Every running copy picks it up within minutes.
